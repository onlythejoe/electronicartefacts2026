Mandate

VASTE is the primary proprietary runtime developed within Electronic Artefacts. It explores graph architectures, contextual execution, identity systems, simulation frameworks, knowledge structures and modular world construction.

The program is not presented only as research material. It is a technical asset that can be evaluated, piloted, licensed or used as the foundation for implementation work when a partner needs graph-shaped infrastructure, identity-aware workflows or knowledge-system architecture.

Commercial access

VASTE is available through qualified commercial conversations rather than open public download. The access model is designed to expose the right level of code, documentation and architecture context without treating a proprietary runtime as a generic package.

Typical access paths include:

  • Repository review for qualified technical buyers or partners.
  • Guided architecture review for platform, product or institutional teams.
  • Scoped pilot around one concrete graph, identity, workflow or knowledge-system use case.
  • Implementation work where Electronic Artefacts adapts VASTE concepts to a production surface.
  • Licensing or partnership discussion when the runtime logic is intended to support a longer product line.

Repository access requests should include the target use case, organization or practice, GitHub username, requested access window, commercial context and any NDA or licensing constraints. The fastest route is to request access by email: Request VASTE repository access.

What can be evaluated

A qualified VASTE review can cover the technical and strategic pieces that matter before a pilot or license decision:

  • The entity and relationship model used to structure graph records.
  • Identity, permissions and context boundaries.
  • Event propagation and projection patterns.
  • Public-surface integration and route design.
  • Knowledge graph publication, search and semantic identifiers.
  • Fit with a buyer’s existing stack, product roadmap or institutional constraints.

Research foundation

The program operationalizes Runtime Theory. It treats entities, relations, identity and context as executable system concerns rather than only database or interface concerns.

Architecture

The current architecture is organized around addressable graph entities, typed relationships, contextual identity and event propagation. Public applications can build specialized surfaces while retaining shared runtime semantics.

The program is structured around several operating layers:

  • Graph entity layer: canonical records, typed references, routeable identifiers and semantic publication.
  • Relationship layer: explicit predicates, bidirectional context and local graph views.
  • Identity and permission layer: boundaries for users, organizations, maintainers, access and visibility.
  • Event execution layer: state changes, propagation and system responses.
  • Projection layer: public pages, product surfaces, search documents, feeds and external interfaces.
  • Commercial access layer: documentation, repository review, pilot boundaries and implementation planning.

Buyer fit

VASTE is most relevant when the product problem is structural rather than cosmetic. It fits teams that need to model people, artefacts, rights, evidence, histories, workflows, projects, institutions or knowledge nodes as connected and evolving entities.

Strong fits include:

  • Knowledge platforms and cultural infrastructure.
  • Archives, catalogues, foundations, collections and institutions.
  • Product teams building permissioned graph workflows.
  • Studios that need reusable architecture across many public surfaces.
  • Research partners investigating contextual execution, identity and simulation.

Weak fits include one-off landing pages, commodity ecommerce, short-term marketing microsites or projects where a conventional CMS is already sufficient.

Lineage

VASTE emerged from ARCA, an earlier system concerned with digital assets, traceability and certification. The scope moved from a specialized asset platform toward generalized execution and information structures.

Implementations

Vestiges is the flagship public product currently used to demonstrate the runtime in a concrete knowledge and economic context. It applies the graph model to culture, craft and human know-how.

Other Electronic Artefacts programs and projects inherit pieces of the same operating model: structured records, reusable publication surfaces, explicit lineage, scoped access and implementation-ready architecture.

Engagement formats

VASTE can enter a collaboration through four practical formats:

  • Technical review: evaluate repository structure, runtime assumptions and fit.
  • Pilot: define one bounded use case, implementation target and success criteria.
  • Build: use VASTE patterns as the foundation for a product, archive, platform or internal system.
  • License: negotiate the use of runtime logic, modules or architectural patterns inside a broader product or institutional context.

Each format should end with an explicit decision: no fit, further research, paid pilot, implementation scope, licensing track or partnership roadmap.

Status and limitations

VASTE remains in active development. Architectural claims are published where they are stable enough to explain, while implementation details and performance characteristics remain subject to revision.

Private repository access does not imply unrestricted redistribution, resale, sublicensing or production use. Those rights must be discussed explicitly before any commercial deployment.
